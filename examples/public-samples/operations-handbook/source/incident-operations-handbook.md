# Incident Operations Handbook

This fictional handbook is a public KnowMesh fixture for Expert SDK tests.

## Policy Scope

Published operations knowledge assets must have an accountable source owner, a weekly review cadence, rollback-ready releases, and cited answers. A source owner confirms whether a document stays active, enters review, or becomes archived.

## Procedure

When an incident response note changes, the owner reviews the affected procedure, checks the evidence requirement, publishes a draft release, and keeps the previous active release available for rollback.

## Roles

The source owner approves content scope. The release operator publishes the knowledge asset only after citation checks pass. The reviewer confirms that feedback items become maintenance work.

## Review Cadence

Operations handbooks use a weekly review cadence. Emergency changes can be reviewed sooner, but they still need cited evidence and a rollback path.

## Rollback Rule

If a published answer cannot cite the current source, KnowMesh should stop treating that answer as usable and roll back to the previous active release after review.

## Evidence Requirement

Every answer about this handbook must cite the source document and page or section. A question without supporting evidence must return a no-answer state.
