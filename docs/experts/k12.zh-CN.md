# KnowMesh Expert - K12

[English](k12.en.md) | [文档中心](../README.md) | [当前设计](../current-design.md)

K12 是 KnowMesh 的第一个强化 Expert 场景。它不是把 KnowMesh 变成教育专用产品，而是证明 KnowMesh Core 可以在保持本地优先、SQLite-first、可审计的前提下，加载领域结构、对象类型、查询路由和评测门槛。

## 它解决什么问题

教材资料不是普通 PDF。用户会问“第三单元第一课是什么”“这课有哪些词语”“这个公式对应哪些练习”“这页出处在哪里”，也会跨册、跨出版社比较。普通切块问答很容易漏掉目录、页码、单元、课文顺序和越界拒答。

K12 Expert 的目标是把教材文件夹编译成有结构的知识资产：

- 目录、单元、课文、栏目、正文、词表、知识点、公式、表格、图、例题、练习、答案解析、实验、活动和引用锚点。
- 查询结果必须能回到教材、页码、章节或结构锚点。
- 问题超出当前知识库拥有的学段、学科、年级、册次、出版社或教材范围时，必须在检索前拒绝。

## Query Runtime 边界

K12 查询必须走统一的 Query Runtime，而不是 UI 专用分支。目录、单元、课文、页码、词表、公式和练习问题优先走结构或对象路由；越界问题在检索前拒答；可用答案必须返回来源、页码或结构锚点引用。Integration API、Web Console 和 SDK 看到的是同一套状态、引用、质量检查和反馈入口。

## 工程边界

K12 Expert 通过 `src/local-service/expert-registry.mjs` 注册，并绑定 `textbook-cn-k12` 模板。General 模板不会加载 K12 处理器。

主要文件：

- `src/experts/k12/template.json`：Expert 元数据、能力清单和质量目标。
- `src/experts/k12/schema.json`：K12 对象类型、关系类型和少量历史别名映射。
- `src/local-service/k12-object-contract.mjs`：对象和关系类型规范化入口。
- `src/local-service/k12-structure-readiness.mjs`：目录、单元、课文和结构查询 readiness。
- `src/local-service/k12-query-router.mjs`：K12 结构和对象查询路由。
- `src/local-service/k12-evaluation-runner.mjs`：必跑评测集和 catalog 评测执行。
- `src/local-service/k12-evaluation-manifest.mjs`：评测结果、质量目标和风险摘要。

## 质量门槛

K12 Expert 目前使用这些最低门槛：

| Gate | Target |
| --- | --- |
| TOC completeness | `>= 0.95`，至少 95% 课文要有目录结构覆盖 |
| Out-of-scope refusal | `1.0`，越界问题必须 100% 拒答 |
| Citation-bearing usable answers | `>= 0.85`，可用答案至少 85% 带引用 |
| Display serialization errors | `0`，可展示响应不能序列化失败 |

这些目标在 API 输出中可审计，而不是只写在文档里。结构 readiness 会输出 `tocCompletenessTarget`、`tocCompletenessRate` 和 `requiredGates`；评测 manifest 会输出 `qualityTargets`、`categoryRates` 和 `requiredGates`。

## 必跑评测类型

K12 评测集覆盖：

- 目录查询
- 单元课文查询
- 词表查询
- 习作 / 口语交际查询
- 数学概念查询
- 数学例题 / 练习查询
- 英语单元主题
- 英语词汇
- 科学实验
- 页码引用
- 跨册比较
- 出版社比较
- 越界拒答
- 无答案行为

评测结果只保存状态、分数、路由意图、引用数量和风险码，不保存原始题目正文或期望答案正文。

## 贡献方向

适合贡献的方向包括：

- 为更多学科补对象抽取规则。
- 为教材目录、页码、栏目的复杂版式补解析 fixture。
- 改进 K12 查询路由和 refusal 规则。
- 增加真实但可公开的合成教材 fixture。
- 为未来 Expert 作者文档沉淀模板、schema 和评测写法。

不要提交教材版权内容、未授权扫描件、学生隐私、教师资料或私有机构数据。
