# KnowMesh Expert - K12

[中文](k12.zh-CN.md) | [Documentation](../README.en.md) | [Current Design](../current-design.md)

K12 is KnowMesh's first strengthened Expert scenario. It does not make KnowMesh an education-only product. It proves that KnowMesh Core can stay local-first, SQLite-first, and auditable while loading domain structure, object types, query routing, and evaluation gates.

## What It Solves

Textbooks are not generic PDFs. Users ask questions such as "what is the first lesson in unit three", "which vocabulary appears in this lesson", "which exercises use this formula", and "where is the cited page". They also compare volumes or publishers. Plain chunk-and-ask workflows often lose table-of-contents structure, page anchors, unit order, lesson order, and strict refusal behavior.

The K12 Expert compiles textbook folders into structured knowledge assets:

- Books, units, lessons, sections, columns, texts, vocabulary, knowledge points, formulas, tables, figures, examples, exercises, answer explanations, experiments, activities, and citation anchors.
- Query results must trace back to a book, page, section, or structure anchor.
- Questions outside the owned stage, subject, grade, volume, publisher, edition, or textbook scope must be refused before retrieval.

## Query Runtime Boundary

K12 questions must use the shared Query Runtime, not a UI-only branch. Table-of-contents, unit, lesson, page, vocabulary, formula, and exercise questions prefer structure or object routes; out-of-scope questions refuse before retrieval; usable answers must return source, page, or structure-anchor citations. The Integration API, Web Console, and SDK see the same statuses, citations, quality checks, and feedback actions.

## Engineering Boundary

The K12 Expert is registered in `src/local-service/expert-registry.mjs` and is bound to the `textbook-cn-k12` template. The general template does not load K12 processors.

Primary files:

- `src/experts/k12/template.json`: Expert metadata, capabilities, and quality targets.
- `src/experts/k12/schema.json`: K12 object types, relation types, and narrow historical alias mapping.
- `src/local-service/k12-object-contract.mjs`: object and relation type normalization.
- `src/local-service/k12-structure-readiness.mjs`: TOC, unit, lesson, and structure-query readiness.
- `src/local-service/k12-query-router.mjs`: K12 structure and object query routing.
- `src/local-service/k12-evaluation-runner.mjs`: required evaluation set and catalog evaluation runner.
- `src/local-service/k12-evaluation-manifest.mjs`: evaluation results, quality targets, and risk summary.

## Quality Gates

The current minimum gates are:

| Gate | Target |
| --- | --- |
| TOC completeness | `>= 0.95`, at least 95% lesson coverage by TOC structure |
| Out-of-scope refusal | `1.0`, out-of-scope questions must be refused 100% of the time |
| Citation-bearing usable answers | `>= 0.85`, at least 85% usable answers carry citations |
| Display serialization errors | `0`, displayable responses must not fail serialization |

These targets are auditable through API output. Structure readiness returns `tocCompletenessTarget`, `tocCompletenessRate`, and `requiredGates`; the evaluation manifest returns `qualityTargets`, `categoryRates`, and `requiredGates`.

## Required Evaluation Categories

The K12 evaluation set covers:

- TOC lookup
- Unit lesson lookup
- Vocabulary lookup
- Writing / oral communication lookup
- Math concept lookup
- Math example / exercise lookup
- English unit theme
- English vocabulary
- Science experiment
- Page citation
- Cross-volume comparison
- Publisher comparison
- Out-of-scope refusal
- No-answer behavior

Evaluation results store status, scores, route intent, citation count, and risk codes. They do not store raw question text or expected answer text in result details.

## Contribution Ideas

Good contribution areas include:

- Object extraction rules for more subjects.
- Fixtures for complex textbook TOC, page, and column layouts.
- Better K12 query routing and refusal rules.
- Public synthetic textbook fixtures.
- Future Expert authoring docs for templates, schemas, and evaluation sets.

Do not commit copyrighted textbook content, unauthorized scans, student privacy data, teacher records, or private institution data.
